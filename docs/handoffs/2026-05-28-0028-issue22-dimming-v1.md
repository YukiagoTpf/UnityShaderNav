## 3. 关键决策
| # | 决策 | 理由 | 被排除的方案 |
|---|------|------|-------------|
| D1 | 客户端 decoration 渲染暗显 | 只有 decoration 能按区域(含注释/空行/嵌套指令)降透明度 | semantic-token modifier:无法整段降透明度 |
| D2 | pull 式自定义 LSP request `unityShaderNav/inactiveRegions` | 复用 semanticTokens handler 模式 + RequestSuspender,改动局部 | server push(如 `unityShaderNav/mode`):要侵入 reindex 管线 |
| D3 | 四值逻辑 `TRUE/FALSE/VARIANT/UNKNOWN`,且 **UNKNOWN 优先于 VARIANT** | 偏向「不误暗显」;未知宏可能由 include 提供 | UNKNOWN/VARIANT 同级:会 false-dim `VARIANT‖UNKNOWN` |
| D4 | 本地 `#undef X` 视为权威 `FALSE` | `#undef` 覆盖一切来源,验收项明确要求识别 `#undef` | `#undef`→UNKNOWN(原计划):太保守,与验收冲突 |
| D5 | `.shader`:variant 全文件收集 + program 块由前序 HLSLINCLUDE 种子注入 define | 对齐 ADR-0001「include 符号对所有 Pass 可见」 | 每块完全独立分析:漏掉 HLSLINCLUDE→Pass 的 pragma/define 上下文 |
| D6 | 协议返回 `{range, reason}[]` 且带 `textDocument.version` | reason 留待未来拆样式;version 供 client 丢弃过期响应 | 裸 `Range[]`:丢 reason;无 version:快速编辑时旧响应覆盖新装饰 |

## 4. 失败路径
| 尝试 | 失败原因 | 教训 |
|------|---------|------|
| 本地 `/codex:review` 跑 review | 共享 Codex runtime 的沙箱 shell 全部 spawn 失败(`pwsh.exe ... exit -1`),只能改用 GitHub MCP 兜底,本地 review 未出有效结论 | 本地共享 runtime 的 shell 不可靠;review 意见最终另行获得(见 commit 3eeb985)。需要时换 `/codex:adversarial-review` 或在别处跑 |
| 原计划 `#undef`→UNKNOWN | 与 issue 验收「识别 file-local `#undef` state」冲突 | `#undef` 在本地流里是确定信息,应判 FALSE |
| 原计划 and/or 表 VARIANT 优先 | `VARIANT‖UNKNOWN` 会 false-dim 实际可能激活的分支 | 保守 = 不确定时保持可见,UNKNOWN 必须压过 VARIANT |
| 原计划每块独立分析 | 违背 ADR-0001 的 HLSLINCLUDE 可见性模型 | 同文件 include 块的 pragma/define 要传递给后续 program 块 |
